# Security Policy

## Supported Versions

We currently provide security updates for the latest stable release.

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please do **not** open a public issue.

Instead, report it privately via email:

**hs0714@qq.com**

You can expect:

- **Acknowledgment** within 48 hours of your report
- **Status update** within 5 business days
- **Fix timeline** communicated once the issue is assessed

## What to Include

To help us respond quickly, please include:
- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Scope

The following are in scope:
- Source code in this repository
- Build pipeline (CI/CD)
- Dependencies with known CVEs

Out of scope:
- Third-party API services used by the app
- User's local environment security

## Responsible Disclosure

We ask that you:
- Allow us reasonable time to fix the issue before disclosure
- Do not exploit the vulnerability
- Act in good faith to improve security

## Recognition

We thank security researchers who help us improve the project. With your permission, we will acknowledge your contribution in release notes.

Thank you for helping keep Rapid Code and its users safe.
